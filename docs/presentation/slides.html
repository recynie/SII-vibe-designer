Vibe Design

多智能体协同设计系统

基于 opencode harness 的命令行设计系统。用户输入一句自然语言 brief，系统自主完成需求理解、品牌规范推导、视觉产物生成、机器校验与主观评审，输出完整品牌视觉交付。

4 Agents·3 Skills·5 Python Tools·E2E Pipeline
opencode + GPT-5.5 + gpt-image-2 2026.05

系统设计

02 / 05

五项设计原则

Harness 不重写，只扩展

opencode 提供调度/上下文/工具调用；本项目只补齐设计能力层

文件即上下文契约

Agent 间通过 facts.md / brand-spec.md / deliverables.md 传递决策，不依赖隐式记忆

先确定交付物，再并行执行

Researcher 按 brief 选择 1-5 个交付物；Planner 只按清单调度

机器硬门槛 + 主观评审分层

validate.py 做可靠机器检查；Critic 读实物做 5 维主观评分

图像后端对 Agent 透明

统一 CLI 接口，开发/正式后端切换不改 agent prompt

关键实现细节

api.toml 统一凭据路由，[active] 切换 LLM/Image 后端
gen_image.py 双后端（MiniMax/OpenAI），并行候选，自动格式检测
html_screenshot.py Playwright Chromium 渲染 HTML→PNG
validate.py 字族合规硬门槛，可扩展检查管线

Critic 评审机制

5 维评分体系（每项 0-5 分）
调性体现
视觉气质
单件构图
信息层级
任务完成度
通过条件：总分 ≥ 18/25 且无单项 ≤ 2

迭代策略

v1 Designer 首次生成 → Critic 评审 v2 未通过 → Designer 根据 review 修改 prompt 重出 v3 仍未通过 → escalate.md，向用户提问
E2E 测试：uv run e2e-test --brief "..."
idle-based timeout + 进度心跳 + final.md 检测
