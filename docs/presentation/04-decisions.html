04 / 05 — DECISIONS
01

串行调度，不并行

Scheduling Discipline

问题：并行调度多个 subagent 时，reasoning 模型偶发长时挂起（5+ 分钟无产出）。

决策：Planner 一次只 @ 一个 subagent，等返回再调下一个。牺牲理论并行度，换取可预测的执行。切到 GPT-5.5 后挂起未再复现，但规定保留——race 风险始终存在。
02

文件即契约

Context Management

问题：subagent 拥有独立上下文，无法直接通信。如何传递设计意图和评审结果？

决策：所有信息通过文件传递。Researcher 写 brief.md + brand-spec.md，Designer 读文件出物，Critic 读实物 + prompt 打分写 review.md。每个 agent 读写约定明确，信息无歧义。
03

Critic 同时审图与 prompt

Review Innovation

问题：很多设计问题的根因在 prompt，不在产出本身。只看图无法给出有效改进建议。

决策：Designer 存盘 v?.prompt.txt，Critic 同时审阅图像和 prompt → 改进建议分三类：改 prompt（如去掉 generic 词）/ 改版式（如调间距）/ 换素材。分层建议让 Designer 精准修改。
04

双图像后端 · LLM 无感

Tool Architecture

问题：开发迭代需要便宜快速的图像模型，正式交付需要高质量模型。切换不应影响 agent 逻辑。

决策：gen_image.py 统一 CLI，后端由 api.toml [active].image 配置路由。Designer prompt 只写 "调 gen_image"，不感知后端。开发用 minimax image-01 省钱，交付用 gpt-image-2 高质。
05

反 AI slop · 一票否决

Quality Gate

问题：AI 生成设计有典型通病——紫渐变、emoji、Inter 当 display、通用模板拼贴，辨识度为零。

决策：Critic 5 维度中"反 slop"项 ≤ 4 分则整体不通过。具体禁区写进 prompt：紫渐变、emoji 图标、SVG 画人物、generic 拼贴卡片。品味是硬约束，不是建议。
06

内容审查可恢复

Error Handling

问题：朱家角 brief 触发 MiniMax 服务端内容审查（1027），整个 designer 子流程被拦。

决策：Designer 触发审查时自动切换到纯英文 prompt + hex+色名双标（如 "sage green #4A7C59"）。Planner prompt 保留 1027 fallback 分支——不是绕过审查，是改写表达方式后重试。
