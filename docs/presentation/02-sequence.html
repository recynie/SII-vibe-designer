关键设计纪律

① 串行调度，不并行

实测：MiniMax-M2 的 reasoning 模式在 subagent 并行调度下会出现长时挂起。
规则：一次只 @ 一个 subagent，等返回再调下一个。

② 文件即契约，不直接通信

subagent 间不互相 @；信息只通过 brief.md / brand-spec.md / v?.review.md 传递。Critic 永远基于已写盘的实物打分，不预测 final。

③ 双图像后端 LLM 无感

Designer prompt 不挑后端，tools/gen_image.py 读 DESIGN_IMAGE_BACKEND 路由：
开发 minimax 省钱，交付 openai gpt-image-2 高质。

④ Critic 同时审图与 prompt

很多 slop 根因在 prompt（如 generic 词、缺禁忌项）。Critic 看 prompt 才能给出"改 prompt / 改版式 / 换素材"分层建议。

⑤ 反 AI slop 是一票否决

5 维度评分中"反 slop"项 ≤4 分则整体不通过：紫渐变、emoji 图标、Inter 当 display、SVG 画人、generic 拼贴卡片均归零。

⑥ 内容审查可恢复

朱家角 brief 触发 MiniMax output new_sensitive 1027，整个 designer 子流程被服务端拦。
修复：designer 触发审查时切换到纯英文 prompt + hex+色名（"sage green #4A7C59"），实测一次跑通。
