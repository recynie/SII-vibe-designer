核心思路

Design Philosophy

不重写 harness，只做领域扩展。opencode 提供调度、上下文隔离、工具调用、错误处理； 我们只写设计领域的 agent prompt + 领域工具 + 入口命令。 用 /design <brief> 一键启动全流程。

题目覆盖

Requirements
  • 至少三个智能体 + 独立上下文 — 实现 4 个（多 Researcher）
  • 必要的设计工具 — gen_image 双后端 + html_screenshot
  • Agent Harness 框架 — opencode 1.4 + 自定义扩展
  • 任务调度 / 信息传递 / 迭代优化 — Planner 串行调度 + Critic 闭环
  • 端到端交付 — 创智学院完整品牌设计，12 分钟全自主
