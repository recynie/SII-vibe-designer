实训赛题成果

Vibe Design 是一个基于 opencode harness 构建的多智能体协同设计系统。系统接收自然语言设计 brief，自动完成事实调研、品牌规范制定、交付物规划、视觉生成、机器校验、视觉评审与迭代修正，最终产出完整的设计交付结果。

核心设计思路：将认知冲突外化为 agent 角色边界。系统包含四个 agent——Planner 负责流程调度与容错，Researcher 负责事实调研与设计范围定义，Designer 负责视觉产物生成，Critic 负责机器校验与视觉评审。每个 agent 拥有独立上下文和精确的权限边界（如仅 Researcher 可 webfetch），从根本上切断了单 agent 场景下的自我辩护和事实编造路径。

Agent 间通信采用文件协议：所有关键状态落盘为 Markdown 文件，不依赖隐式聊天记忆。这一选择通用于所有 LLM，便于人类审阅和定位问题，天然支持版本管理和权限控制。上游文件通过独立的 CLI 校验工具进行格式化检查，保证 agent 间信息传递的确定性。

工具层采用 CLI 接口而非 opencode Tool Call，降低了调试复杂度并使中间结果完全可见。核心工具包括：gen_image.py（双后端文生图，支持候选图并行择优）、html_screenshot.py（HTML 渲染为 PNG，使 agent 能"看到"排版结果）、validate.py（聚合校验，HTML 字族硬门槛 + 文件存在性检查）。系统支持图像类和 HTML 排版类两种产物类型，通过渲染截图使排版问题可被视觉评审发现和迭代修复。

在最具代表性的实验中，系统为上海创智学院设计了两组不同风格的文创产品，覆盖 12 个产品类别，最终产出 101 个文件（50 张 PNG、36 份 prompt 记录、5 份 review、2 份 HTML），持续运行约 2 小时。期间系统自主处理了网络波动和 API 异常，完成了从字族硬门槛失败到修复通过再到视觉评审通过的完整迭代闭环。

系统的核心贡献在于验证了一条可执行、可复盘、可扩展的 AI For Design 工作流：将设计流程分解为独立的认知角色，用文件协议承载 agent 间信息交换，用 CLI 工具支持自动化校验，用 Planner 的灵活调度实现鲁棒的长时间自主运行。
