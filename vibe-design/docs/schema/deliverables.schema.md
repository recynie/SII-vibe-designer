# Schema · deliverables.md

`deliverables.md` 是 researcher 在 facts.md / brand-spec.md 之后产出的第三份结构化文件,给 planner 当调度清单。**四段式**结构:显式 / 隐式 / 拒绝 / 决策依据。

## 顶层结构

```markdown
# Deliverables · <主题名>

## 显式
- <名称> | <一句话规格>

## 隐式
- <名称> | <一句话规格 + 推断理由>

## 拒绝
- <名称> | <为什么不做>

## 决策依据
- 显式条目来自 brief 第 N 句 / facts.md「<片段>」
- 隐式条目的依据 / 拒绝条目的依据
```

## 行级语法

### 显式 / 隐式 段

#### 单产物条目(默认)

每行必须形如:

```
- <名称> | <规格说明>
```

- `<名称>` 非空、不含 `|` 字符
- `<规格说明>` 一句话,说明用途、尺寸、关键约束(不要写做法)

#### 多子产物条目(sub-items)

当一个交付物天然包含**多个形态各异的子产物**时(例如「品牌文创设计」包含 logo + 多个文创效果图),使用主条目 + 缩进子条目的语法:

```
- <主名称> | <整体规格说明>
  - <子产物名> | <子规格>
  - <子产物名> | <子规格>
  ...
```

语法规则:

- 主条目格式与单产物条目**完全相同**(`- <名称> | <规格>`)
- 子条目以 **2 空格缩进** + `- ` 开头,格式为 `  - <子产物名> | <子规格>`
- 子产物数量:**≥ 2 条**(否则用单产物条目即可),**≤ 6 条**
- `<子产物名>` 非空、不含 `|`,在同一主条目内唯一
- `<子规格>` 一句话,说明形态(PNG 效果图 / HTML / 纯文案 md)、尺寸、关键约束
- 子产物间可以有**依赖关系**:后出现的子产物可以引用前面子产物的产出(如「基于 logo 子产物生成 T 恤效果图」);researcher 须在子规格中用自然语言说明依赖

**何时用 sub-items**:一个交付物需要产出 ≥ 2 个独立文件、且这些文件归属同一个设计主题时。典型场景:
- 品牌文创设计:logo + T恤效果图 + 帆布袋效果图 + 马克杯效果图
- 品牌视觉规范页:规范文档 HTML + 标准色板图 + 字体示例图
- 宣传物料套件:主海报 + 横幅 banner + 社交媒体封面

**何时不用 sub-items**:
- 交付物只有一个文件 → 用单产物条目
- 多个独立交付物(logo 和海报是两件事)→ 分成两个单产物条目

**计数规则**:一个含 sub-items 的主条目计为 **1 条**交付物(不按子产物数拆开计数)。显式+隐式合计 ≤ 5 条的限制对主条目计数。

### 拒绝段

每行必须形如:

```
- <名称> | <理由>
```

不带 mode(拒绝项不会被生产)。理由不能为空。

### 决策依据段

自由文本,不强校验,但必须非空(用于事后审计)。

## 校验规则(`tools/validate_deliverables.py`)

1. 第一行必须以 `# Deliverables` 开头
2. 必须包含四个一级章节:`## 显式` `## 隐式` `## 拒绝` `## 决策依据`(顺序固定)
3. 显式段至少 1 条
4. 显式 / 隐式段**主条目**(非缩进行)必须能拆分为 2 段(`<名称>` / `<规格>`)
5. **子条目**(2 空格缩进行)必须能拆分为 2 段(`<子产物名>` / `<子规格>`),且其前面最近的非缩进行必须是合法主条目
6. 子条目数量:同一主条目下 ≥ 2 且 ≤ 6;若只有 1 个子条目则报错(应改用单产物条目)
7. 子产物名在同一主条目内唯一
8. 拒绝段每行必须能拆分为 2 段(`<名称>` / `<理由>`)
9. 决策依据段必须有至少 1 条 `- ` 起头的行
10. 显式 + 隐式段中,**主条目**名称去重(不接受同名两条)
11. **隐式段 ≤ 2 条**(按主条目计);超过则报错
12. **显式 + 隐式合计 ≤ 5 条**(按主条目计);超过则报错(次要物料移到拒绝段并写"超出 5 件上限")

## 示例(合规,单产物条目)

```markdown
# Deliverables · 创智学院

## 显式
- 招生 KV | 1024×1024 PNG 效果图，学院建筑 + 学生场景，摄影感，体现学术活力
- 招生宣传海报 | 1200×1600 HTML 海报 + 渲染 PNG，竖版四层结构，招生信息 + CTA，页眉使用 assets/sii-logo.svg
- 文创帆布袋效果图 | 1024×1024 PNG，白色帆布袋正面印有 logo + slogan，柔和棚拍
- 品牌文案 | slogan（14 字内）× 3 变体 + 80-120 字简介，纯 markdown

## 隐式
- 品牌一段简介 | 80-120 字第三人称;用户未明说但宣传场景必备

## 拒绝
- 品牌视觉规范页 | 本 run 聚焦招生场景，规范页另行处理
- 短视频脚本 | brief 未提及视频媒介，超出本 run 平面设计范围

## 决策依据
- 显式条目来自 brief「请为创智学院做一套品牌形象设计」+ 招生场景分析
- 招生 KV 用 gen_image 出摄影感场景图，不走 HTML 排版
- 招生海报走 HTML 排版，承载具体招生信息 + CTA
- 文创帆布袋是校园高可见度实用物件，补充品牌落地感
- 简介属隐式：宣传海报的副本必备，但 brief 未点名
```

## 示例(合规,含 sub-items)

```markdown
# Deliverables · 创智学院

## 显式
- 品牌文创设计 | 基于品牌视觉系统,产出 logo + 多个文创载体效果图
  - Logo 主标志 | 1024×1024 PNG 效果图,扁平化矢量风
  - T恤效果图 | 1024×1024 PNG,白色 T恤正面印有 logo,活动场景布局
  - 帆布袋效果图 | 1024×1024 PNG,logo + slogan 印在帆布袋正面
  - 马克杯效果图 | 1024×1024 PNG,白瓷杯侧面印有 logo
- 品牌文案 | slogan(14 字内)+ 80-120 字简介 + 定位语,纯 md
- 主视觉海报 | 1200×1600 PNG 效果图,承载品牌门面
- 官网首页 Mockup | 1440×2400 HTML 落地页 + 渲染 PNG

## 隐式
- 品牌视觉规范页 | A4 HTML 规范文档,展示标准色板/字体/用法

## 拒绝
- 短视频脚本 | brief 未提及视频媒介,超出本 run 平面设计范围
- 名片设计 | brief 未点名,属于品牌延展物料

## 决策依据
- 「品牌文创设计」整合了 logo + 文创载体效果图,因为文创载体依赖 logo,属同一设计主题
- T恤/帆布袋/马克杯是学院场景常见文创载体,补充品牌落地感
- 显式 4 条 + 隐式 1 条 = 5,未超上限
```

## 示例（违规，多类）

```markdown
# Deliverables · X

## 显式
- Logo 缺规格                           ← 缺 | 分隔,无规格说明
- 海报                                   ← 缺 | 分隔,无规格说明
- Logo | 同名重复                        ← 与第一条同名
- 文创套件 | 文创设计
  - T恤 | 只有一个子产物     ← sub-items 数量 < 2

## 隐式
（空段）

## 拒绝
- 视频                                 ← 理由空

## 决策依据
（空）
```

校验脚本必须分别报告：分隔符缺失、同名重复、显式段空、拒绝段无理由、决策依据空、sub-items 数量不足。
