# Schema · deliverables.md

`deliverables.md` 是 researcher 在 facts.md / brand-spec.md 之后产出的第三份结构化文件，给 planner 当调度清单。结构保持简单：**显式 / 隐式 / 拒绝 / 决策依据**。

researcher 下载到 `assets/` 的文件是原始材料；某个交付物需要它，就在规格里直接写 `assets/<filename>`。

## 顶层结构

```markdown
# Deliverables · <主题名>

## 显式
- <名称> | <一句话规格>
- <名称> | <一句话规格；必要时引用 assets/<filename>>

## 隐式
- <名称> | <一句话规格 + 推断理由>

## 拒绝
- <名称> | <为什么不做>

## 决策依据
- 显式条目来自 brief 第 N 句 / facts.md「<片段>」
- 隐式条目的依据 / 拒绝条目的依据
```

## 行级语法

### 显式 / 隐式 段

每行必须形如：

```markdown
- <名称> | <规格说明>
```

- `<名称>` 非空、不含 `|` 字符
- `<规格说明>` 一句话，说明用途、尺寸、产物形态、关键约束
- 如果需要使用现有素材，规格里直接引用 `assets/<filename>`
- 不在行内写工具链模式；designer 根据产物形态和素材路径选择工具

### 拒绝段

每行必须形如：

```markdown
- <名称> | <理由>
```

拒绝项不会被生产。理由不能为空。

### 决策依据段

自由文本，不强校验，但必须非空（用于事后审计）。

## 校验规则（`tools/validate_deliverables.py`）

1. 第一行必须以 `# Deliverables` 开头
2. 必须包含四个一级章节：`## 显式` `## 隐式` `## 拒绝` `## 决策依据`（顺序固定）
3. 显式段至少 1 条
4. 显式 / 隐式段每行必须能拆分为 2 段（`<名称>` / `<规格>`）
5. 拒绝段每行必须能拆分为 2 段（`<名称>` / `<理由>`）
6. 决策依据段必须有至少 1 条 `- ` 起头的行
7. 显式 + 隐式段中，名称去重（不接受同名两条）
8. **隐式段 ≤ 2 条**；超过则报错
9. **显式 + 隐式合计 ≤ 5 条**；超过则报错（次要物料移到拒绝段并写"超出 5 件上限"）

## 示例（合规）

```markdown
# Deliverables · 创智学院

## 显式
- 招生 KV | 1024×1024 PNG 效果图，学院建筑 + 学生场景，摄影感，体现学术活力
- 招生宣传海报 | 1200×1600 HTML 海报 + 渲染 PNG，竖版四层结构，招生信息 + CTA，页眉使用 assets/sii-logo.svg
- 文创帆布袋效果图 | 1024×1024 PNG，白色帆布袋正面印有 logo + slogan，柔和棚拍
- 品牌文案 | slogan（14 字内）× 3 变体 + 80-120 字简介，纯 markdown

## 隐式
- 品牌一段简介 | 80-120 字第三人称；用户未明说但宣传场景必备

## 拒绝
- 品牌视觉规范页 | 本 run 聚焦招生场景，规范页另行处理
- 短视频脚本 | brief 未提及视频媒介，超出本 run 平面设计范围

## 决策依据
- 显式条目来自 brief「请为创智学院做一套品牌形象设计」+ 招生场景分析
- 招生 KV 用 gen_image 出摄影感场景图，不走 HTML 排版
- 招生海报走 HTML 排版，承载具体招生信息 + CTA
- 文创帆布袋是校园高可见度实用物件，补充品牌落地感
- 简介属隐式：宣传海报的副本必备，但 brief 未点名
```

## 示例（违规，多类）

```markdown
# Deliverables · X

## 显式
- Logo | 缺规格
- 海报 缺分隔符
- Logo | 同名重复

## 隐式
（空段）

## 拒绝
- 视频

## 决策依据
（空）
```

校验脚本必须分别报告：分隔符缺失、显式段空、拒绝段无理由、决策依据空。
